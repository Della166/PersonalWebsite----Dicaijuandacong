# 个人网站开发计划

## 执行策略：分阶段 Subagent 并行

为避免上下文溢出，将 18 个任务拆成 **6 个阶段**，每阶段最多 **4 个 subagent 并行**执行。每个 subagent 只拿到自己需要的上下文。

```
Phase 1: 地基        [1 agent, 顺序]     ← init-project
Phase 2: 配置层      [2 agents, 并行]    ← theme + i18n
Phase 3: 骨架        [1 agent, 顺序]     ← layout + nav + footer
Phase 4: 页面区块    [4 agents, 并行x2]  ← 8个section分两批
Phase 5: 内容系统    [3 agents, 并行]    ← MDX组件 + 媒体管道 + 内容管道
Phase 6: 发布 & 打磨 [3 agents, 并行]    ← CLI工具 + 平台导出 + 动画响应式
```

### Phase 1 — 项目初始化（1 agent）

- 创建 Next.js 项目、安装全部依赖、配置 tsconfig/next.config/tailwind.config
- 创建目录结构骨架（content/ scripts/ templates/ _media/ 等）
- 创建 .gitignore（含 _media/ dist/）、.env.example、README.md

### Phase 2 — 主题 + 国际化（2 agents 并行）

- **Agent A**: Tailwind 暖绿主题配置 + globals.css + CSS 变量 + 字体
- **Agent B**: next-intl 完整配置（routing.ts, request.ts, middleware.ts, messages/en.json, messages/zh.json）

### Phase 3 — 布局骨架（1 agent）

- 根 Layout、Navbar（毛玻璃 + 语言切换 + 锚点导航）、Footer
- 首页 page.tsx 框架（组合所有 section 占位）

### Phase 4 — 页面 Section（4 agents, 分两批）

**批次 A（4 agents 并行）：**

- **Agent 1**: Hero（动画背景 + 标语 + CTA）
- **Agent 2**: About（个人介绍 + 研究方向）
- **Agent 3**: Skills（技能可视化）
- **Agent 4**: Projects（卡片网格 + Tab 过滤）

**批次 B（4 agents 并行）：**

- **Agent 5**: Research（论文卡片）
- **Agent 6**: Blog 系统（MDX 解析 + Notion + 列表页 + 详情页）
- **Agent 7**: Creative（视频作品展示）
- **Agent 8**: Contact（联系方式 + 社交链接）

### Phase 5 — 内容 & 媒体系统（3 agents 并行）

- **Agent A**: MDX 富媒体组件（Bilibili/YouTube/VideoPlayer/ImageGallery/GifPlayer/Callout）
- **Agent B**: 媒体上传管道（R2 client + upload 脚本 + 路径替换逻辑）
- **Agent C**: 内容管道（frontmatter 类型定义 + 模板文件 + content-loader）

### Phase 6 — 发布工具 & 最终打磨（3 agents 并行）

- **Agent A**: CLI 工具（content:new / content:list / content:publish / media:upload）
- **Agent B**: 平台导出器（微信/小红书/抖音）
- **Agent C**: 全站动画打磨 + 响应式适配 + 性能优化

## 技术栈

- **框架**: Next.js 15 (App Router) + TypeScript
- **样式**: Tailwind CSS 4 + CSS 变量（自定义暖绿色主题）
- **动画**: Framer Motion（页面过渡、滚动动画、元素进场效果）
- **国际化**: next-intl（中英双语路由 `/zh` `/en`）
- **博客**: MDX 本地文件 + Notion API（双来源）
- **媒体存储**: Cloudflare R2（S3 兼容，免费 10GB/月）作为 CDN 图床
- **图标**: Lucide React
- **部署**: Vercel

## 色彩方案

温暖、舒适的绿色科技风格——深色底 + 暖绿高亮 + 琥珀点缀：

- **背景**: 深墨绿 `#0a0f0d` 渐变到 `#0d1a14`
- **主色**: 暖鼠尾草绿 `#7fbc8c` / `#a8d5b0`
- **强调色**: 琥珀暖金 `#d4a574`（用于 CTA 和重点标注）
- **文字**: 暖白 `#f0efe8`
- **卡片**: 半透明玻璃拟态（glassmorphism），带绿色色调的 `backdrop-blur`
- **边框/光晕**: 绿色微光 glow 效果，营造科技感

## 页面结构

整站为**单页滚动式**首页 + 独立的博客详情页，导航栏固定顶部，支持平滑锚点跳转。

### 首页各 Section

1. **Hero** - 全屏开场，粒子/网格背景动画，名字 + 一句话定位 + 三个身份标签（全栈开发者 / AI 研究者 / 全能内容创作者），CTA 按钮
2. **About** - 个人介绍，带照片/头像区域，展示求职方向（AI Engineer / ML Engineer）和研究领域（视频生成、图片生成、多模态等）
3. **Skills** - 技能点可视化展示，分类为：AI/ML、工程开发、内容创作三大板块，用进度条或雷达图
4. **Projects** - GitHub 项目展示卡片网格，分类 Tab 过滤（Agent / Web App / 小程序 / 其他），每张卡片含项目名、描述、技术栈标签、GitHub 链接
5. **Research** - 论文展示区域，卡片列表形式，包含标题、会议/期刊、摘要、链接
6. **Blog** - 博客文章预览（从 MDX + Notion 双来源获取），卡片展示最新文章
7. **Creative** - 创作者作品集展示，视频缩略图网格/轮播，展示 AI 短片和驾驶拍摄作品
8. **Contact** - 联系方式 + 社交链接（GitHub、邮箱等）

### 独立页面

- `/blog` - 博客列表页
- `/blog/[slug]` - 博客文章详情页（MDX 渲染）

## 关键设计细节

- **导航栏**: 固定顶部，半透明毛玻璃效果，滚动时出现底部边框光晕；右侧含语言切换按钮（中/EN）
- **Hero 背景**: 使用 Canvas 或 CSS 实现动态网格/粒子效果，绿色光点缓慢飘动，营造科技感
- **卡片**: 统一使用 `GlassCard` 组件，半透明背景 + backdrop-blur + 绿色边框 glow，hover 时有微妙的光晕放大效果
- **滚动动画**: 每个 section 进入视口时触发 Framer Motion 的 fade-in + slide-up 动画
- **响应式**: 移动端优先，所有布局在手机/平板/桌面端均表现良好

## 数据管理

项目数据、技能数据、论文数据以 TypeScript 文件维护在 `src/data/` 下。博客等动态内容通过统一内容加载器（`content-loader.ts`）从 MDX 本地文件 + Notion API 双渠道获取并合并。

## 国际化方案

使用 `next-intl` 的 App Router 集成方案：

- URL 结构: `/en/...` 和 `/zh/...`
- 所有 UI 文案存储在 `messages/en.json` 和 `messages/zh.json`
- 内容文件通过 frontmatter 的 `lang` 字段标识语言，或在同一文件中包含双语内容

## 内容管道系统（Content Pipeline）

### 核心理念

`content/` 目录是你所有内容的**单一数据源**。写一篇 MDX，网站自动更新；运行一条命令，全平台同步。对 AI 助手（如 Open Claw）极度友好——所有操作都是标准化的文件 + CLI。

### 内容工作流（含媒体处理）

```
1. 写 MDX + 把图片/GIF 放入 _media/
          ↓
2. npm run media:upload → 自动上传到 R2 CDN，MDX 中的本地路径替换为 CDN URL
          ↓
3. 删除 _media/ 中的本地文件（可选，_media/ 已被 .gitignore 忽略）
          ↓
4. git push → Vercel 自动部署网站（零操作）
          ↓
5. npm run content:export → 生成全平台格式化内容
          ↓
   微信公众号: 自动通过 API 发布草稿
   小红书: 输出格式化文案 + 封面提示，手动/AI助手粘贴发布
   抖音: 输出标题、标签、描述文案，手动/AI助手发布
```

也可以一步到位：`npm run content:publish [slug]` = 上传媒体 + git push + 导出全平台

### MDX 富媒体组件

在 MDX 文章中可以直接使用以下自定义组件，无需额外 import：

```mdx
## 嵌入 B站视频
<Bilibili bvid="BV1xx411c7mD" />

## 嵌入 YouTube 视频
<YouTube id="dQw4w9WgXcQ" />

## 通用视频播放器（直链 mp4/webm）
<VideoPlayer src="https://cdn.xxx.com/videos/demo.mp4" poster="https://cdn.xxx.com/images/poster.jpg" />

## 图片画廊（灯箱点击放大）
<ImageGallery
  images={[
    { src: "https://cdn.xxx.com/img/shot1.jpg", alt: "镜头1" },
    { src: "https://cdn.xxx.com/img/shot2.jpg", alt: "镜头2" },
    { src: "https://cdn.xxx.com/img/shot3.gif", alt: "GIF效果" },
  ]}
  columns={3}
/>

## GIF 播放器（默认显示首帧静图，点击播放，节省流量）
<GifPlayer src="https://cdn.xxx.com/img/demo.gif" alt="效果演示" />

## 提示框
<Callout type="tip">这是一个实用技巧</Callout>
```

**组件实现要点：**

- `Bilibili` / `YouTube`: 响应式 iframe 嵌入，16:9 比例自适应，懒加载（进入视口时才加载 iframe）
- `VideoPlayer`: 基于 HTML5 `<video>` 标签，支持 poster 封面图、自定义控件皮肤（匹配暖绿主题）
- `ImageGallery`: CSS Grid 网格布局，点击弹出 lightbox 灯箱（用 Framer Motion 动画），支持左右切换、键盘导航
- `GifPlayer`: 默认用静态首帧占位，点击切换为动态 GIF，再次点击暂停；大幅节省流量
- `Callout`: info / warning / tip / danger 四种类型，带图标和主题色边框
- 所有组件在 `components/mdx/index.ts` 统一注册，MDX 渲染时自动注入

### 媒体上传管道（Media Pipeline）

**设计思路**: 本地放文件 → 一键上传到 Cloudflare R2 CDN → MDX 中自动替换为云端 URL → 本地文件可安全删除。

**工作流程：**

```
_media/                          ← 本地暂存区（.gitignore 忽略）
├── sora-film-cover.jpg
├── behind-the-scenes.gif
└── architecture-diagram.png

运行: npm run media:upload

脚本自动执行：
1. 扫描 _media/ 目录中所有文件
2. 计算文件哈希，检查是否已上传（避免重复）
3. 上传到 Cloudflare R2 的 media/{hash}/{filename} 路径
4. 扫描 content/ 下所有 MDX 文件
5. 将其中引用本地路径（如 /_media/sora-film-cover.jpg）替换为 CDN URL
6. 输出上传报告：✅ 3 files uploaded, 2 MDX files updated
7. 本地 _media/ 文件可安全删除（或加 --clean 自动清理）
```

**环境变量（.env.local）：**

```
R2_ACCOUNT_ID=你的Cloudflare账户ID
R2_ACCESS_KEY_ID=R2访问密钥
R2_SECRET_ACCESS_KEY=R2密钥
R2_BUCKET_NAME=personal-website-media
R2_PUBLIC_URL=https://media.yoursite.com  # R2 公开访问域名
```

**R2 选择理由**: 免费额度 10GB 存储 + 每月 1000 万次读取，S3 兼容 API（用 `@aws-sdk/client-s3` 即可），自带 CDN 全球加速，完美适合图床场景。

### CLI 工具

在 `package.json` 中注册以下命令：

- `**npm run content:new**` - 交互式创建新内容，从模板生成 MDX 文件，自动填入日期和元数据
- `**npm run content:list**` - 列出所有内容及其发布状态
- `**npm run media:upload**` - 上传 `_media/` 中的文件到 R2 CDN，自动替换 MDX 中的本地路径
- `**npm run media:upload --clean**` - 上传后自动清理本地文件
- `**npm run content:export [slug] --platform wechat**` - 将指定内容导出为目标平台格式
- `**npm run content:export [slug] --platform all**` - 导出所有平台格式
- `**npm run content:wechat [slug]**` - 通过微信公众号 API 发布为草稿
- `**npm run content:publish [slug]**` - 一步到位：上传媒体 + git commit & push + 导出全平台

所有命令均支持 `--yes` 非交互模式，方便 AI 助手自动调用。

### 各平台导出策略（含媒体适配）

- **微信公众号**（全自动）:
  - MDX → 去除 JSX 组件 → 公众号兼容 HTML（内联样式）
  - 图片：直接引用 R2 CDN URL（公众号支持外链图片）
  - 视频：`<Bilibili>` / `<YouTube>` 组件转为「点击观看视频」链接卡片（公众号不支持 iframe）
  - GIF：保留为 `<img>` 标签
  - 通过官方 API 创建草稿，手机上点确认发布
- **小红书**（半自动）:
  - 提取正文 → 纯文本 + 格式化 hashtag
  - 图片：从 CDN 下载到 `dist/xiaohongshu/images/`，方便批量上传
  - 视频：提取视频链接，提示手动上传视频文件
  - 输出到 `dist/xiaohongshu/`，方便复制粘贴或 AI 助手操作
- **抖音**（半自动）:
  - 提取标题、标签、描述，格式化为抖音风格文案（含热门话题 #标签#）
  - 视频：提示使用对应的视频文件
  - 输出到 `dist/douyin/`

### AI 助手友好设计

- 所有内容为纯文本 MDX 文件，AI 可直接创建和编辑
- CLI 工具全部支持 `--yes` 非交互模式，适合自动化调用
- `_media/` 暂存区 + `npm run media:upload` 让 AI 助手也能处理图片工作流
- `content/` 目录结构清晰，按类型分文件夹，AI 无需猜测
- Frontmatter schema 有 TypeScript 类型定义（`src/types/content.ts`），确保数据一致性
- `npm run content:publish` 一条命令走完全流程，AI 助手只需调用一次
